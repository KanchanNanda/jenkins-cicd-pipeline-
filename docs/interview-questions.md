Jenkins CI/CD Interview Questions:-


Q1.What is CI/CD?
Answer: CI/CD is a software development practice where code integration, testing, and deployment are automated.

Q2. Why do we use Jenkins?
Answer: Jenkins automates the software delivery process by building, testing, and deploying applications.

Q3. What is Jenkinsfile?
Answer: Jenkinsfile is a text file that contains pipeline definitions written as code.


Q4. Difference between CI and CD?
Answer: CI focuses on automatically integrating and testing code changes.
        CD focuses on automatically delivering or deploying applications.

Q5. Why Docker is used in CI/CD?
Answer: Docker provides consistent environments by packaging applications with dependencies.

Q6. What happens when code is pushed to GitHub?
Answer:
1. Jenkins gets the latest code.
2. Pipeline starts.
3. Docker image is created.
4. Container is deployed.

Q7. What is a Docker image?
Answer: A Docker image is a package containing application code, libraries, and dependencies required to run a container.

Q8. What is a Docker container?
Answer: A container is a running instance of a Docker image.

Q9. What is a Jenkins Pipeline?
Answer: A Jenkins Pipeline is an automated workflow containing multiple stages like build, test, and deployment.

Q10. How do you troubleshoot pipeline failures?
Answer:
- Check Jenkins console output.
- Verify Git connection.
- Check Docker service.
- Check permissions.
- Validate Jenkinsfile syntax.
